# @word-kit/core

OOXML-compliant (ECMA-376) `.docx` reading, editing, and generation. Runs in
modern browsers and Node.js with no Node-only runtime dependencies.

See the [repository README](../../README.md) for the project overview and
[`PLAN.md`](../../PLAN.md) for the roadmap.

## Status

Pre-1.0. Public API may shift before v1, but the high-level surface
(`Docx.open` / `Docx.create` / `toUint8Array`) is stable.

## Install

```bash
npm install @word-kit/core
# or
pnpm add @word-kit/core
```

## Quickstart

### Create a document from scratch

```ts
import { Docx, PAGE_SIZE_A4, MARGINS_NORMAL } from "@word-kit/core";

const doc = Docx.create({ paragraphs: [] });

doc.setPageSize(PAGE_SIZE_A4);
doc.setPageMargins(MARGINS_NORMAL);
doc.setCoreProperties({ title: "Quarterly Report", creator: "Yamada" });

doc.addHeader("Confidential");
doc.addFooter("Page");

doc.addStyle({
  type: "paragraph",
  styleId: "Heading",
  name: "Heading",
  basedOn: "Normal",
  bold: true,
  fontSizeHalfPoints: 32,
  color: "1F497D",
});

doc.appendParagraph("Quarterly Report", { style: "Heading" });
doc.appendParagraph("This document was generated by @word-kit/core.");

doc.addBulletList(["Revenue ↑ 12%", "Costs ↓ 3%", "Headcount: 47"]);
doc.addNumberedList(["Plan Q3", "Hire engineers", "Ship v2.0"]);

doc.addTable([
  ["Metric", "Q1", "Q2", "Q3"],
  ["Users", "1,000", "1,500", "2,200"],
]);

doc.appendPageBreak();
doc.addHyperlink("https://example.com/", "External report");

const bytes = await doc.toUint8Array(); // Uint8Array
// Node: writeFileSync("out.docx", bytes)
// Browser: const blob = doc.toBlob(); download via <a download> or saveAs()
```

### Open an existing `.docx` (template) and edit it

```ts
import { Docx } from "@word-kit/core";

// Node
const tpl = Docx.open(readFileSync("template.docx"));

// Browser
// const tpl = await Docx.fromBlob(fileInput.files[0]);

// Run-spanning find / replace. Works even when {{name}} is split across
// multiple <w:r> elements by Word's spell-checker.
tpl.replaceText("{{name}}", "山田太郎");
tpl.replaceText("{{date}}", new Date().toISOString().slice(0, 10));

// Replace across body + headers + footers + comments + footnotes:
tpl.replaceTextEverywhere(/\{\{(\w+)\}\}/g, (m) => values[m.captures[0]!] ?? "");

// Append a paragraph to the bottom of the body:
tpl.appendParagraph("Updated by template fill on " + new Date().toLocaleString());

writeFileSync("out.docx", tpl.toUint8Array());
```

### Read text content

```ts
const doc = Docx.open(bytes);

doc.text; // visible body text (joined paragraphs with \n)
doc.paragraphs; // WmlParagraph[]
doc.tables; // WmlTable[]
doc.stylesPart; // styles.xml AST (undefined if absent)
doc.commentsPart?.comments; // each XML element of a <w:comment>
doc.coreProperties; // { title?, creator?, subject?, ... } from docProps/core.xml

// Inspect a template for placeholders before bulk-replacing:
for (const { partName, matches } of doc.findTextEverywhere(/\{\{\w+\}\}/g)) {
  console.log(
    partName,
    matches.map((m) => m.text),
  );
}
```

### Tracked changes

```ts
doc.acceptAllRevisions(); // keeps inserted text, drops deletions
doc.rejectAllRevisions(); // drops insertions, keeps deletions as plain text
```

### Comments, footnotes, bookmarks, page breaks

```ts
const para = doc.paragraphs[0]!;

doc.addComment(para, { author: "Reviewer", text: "Please confirm." });
doc.addFootnote(para, "Source: internal report");
doc.addEndnote(para, "Glossary entry");

const bookmarkId = doc.addBookmark("ch1", para);
doc.addInternalHyperlink("ch1", "Jump to Chapter 1");

doc.appendPageBreak();
```

### Images

```ts
const imageBytes = readFileSync("logo.png");
doc.addImage(imageBytes, {
  widthEmu: 914400, // 1 inch
  heightEmu: 914400, // 1 inch
  altText: "Company logo",
});
```

## Architecture in one diagram

```
@word-kit/core         ─── public Docx wrapper (open / create / API)
       │
       ├── @word-kit/wml          ─── WordprocessingML AST + parsers + builders
       ├── @word-kit/ooxml-xml    ─── namespace-aware XML parser/serializer
       └── @word-kit/opc          ─── OPC (ZIP + Content Types + rels)
```

The lower layers are reachable via `docx.opc`, `docx.document`,
`docx.stylesPart` etc. for callers that need direct AST access.
