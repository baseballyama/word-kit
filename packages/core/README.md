# @word-kit/core

OOXML-compliant (ECMA-376) `.docx` reading, editing, and generation. Runs in
modern browsers and Node.js with no Node-only runtime dependencies.

See the [repository README](../../README.md) for the project overview and
[`PLAN.md`](../../PLAN.md) for the roadmap.

## Status

Pre-1.0. Public API may shift before v1, but the high-level surface
(`openDocx` / `createDocx` / `toUint8Array`) is stable.

## API shape

Every operation is exposed as a **standalone function** that takes a
`Docx` value as its first argument. `Docx` is a plain data type (not a
class) returned by `createDocx`, `openDocx`, `fromBlob`, or `clone`.
Standalone functions let a bundler tree-shake any operation you don't
import — a minimal `createDocx + appendParagraph + toUint8Array` slice
ships in ~42 KB minified versus ~131 KB for the full surface (verified
in CI).

## Install

```bash
npm install @word-kit/core
# or
pnpm add @word-kit/core
```

## Quickstart

### Create a document from scratch

```ts
import {
  addBulletList,
  addFooter,
  addHeader,
  addHyperlink,
  addNumberedList,
  addStyle,
  addTable,
  appendPageBreak,
  appendParagraph,
  createDocx,
  MARGINS_NORMAL,
  PAGE_SIZE_A4,
  setCoreProperties,
  setPageMargins,
  setPageSize,
  toUint8Array,
} from "@word-kit/core";

const doc = createDocx({ paragraphs: [] });

setPageSize(doc, PAGE_SIZE_A4);
setPageMargins(doc, MARGINS_NORMAL);
setCoreProperties(doc, { title: "Quarterly Report", creator: "Yamada" });

addHeader(doc, "Confidential");
addFooter(doc, "Page");

addStyle(doc, {
  type: "paragraph",
  styleId: "Heading",
  name: "Heading",
  basedOn: "Normal",
  bold: true,
  fontSizeHalfPoints: 32,
  color: "1F497D",
});

appendParagraph(doc, "Quarterly Report", { style: "Heading" });
appendParagraph(doc, "This document was generated by @word-kit/core.");

addBulletList(doc, ["Revenue ↑ 12%", "Costs ↓ 3%", "Headcount: 47"]);
addNumberedList(doc, ["Plan Q3", "Hire engineers", "Ship v2.0"]);

addTable(doc, [
  ["Metric", "Q1", "Q2", "Q3"],
  ["Users", "1,000", "1,500", "2,200"],
]);

appendPageBreak(doc);
addHyperlink(doc, "https://example.com/", "External report");

const bytes = toUint8Array(doc); // Uint8Array
// Node: writeFileSync("out.docx", bytes)
// Browser: const blob = toBlob(doc); download via <a download> or saveAs()
```

### Open an existing `.docx` (template) and edit it

```ts
import {
  appendParagraph,
  fromBlob,
  openDocx,
  replaceText,
  replaceTextEverywhere,
  toUint8Array,
} from "@word-kit/core";

// Node
const tpl = openDocx(readFileSync("template.docx"));

// Browser
// const tpl = await fromBlob(fileInput.files[0]);

// Run-spanning find / replace. Works even when {{name}} is split across
// multiple <w:r> elements by Word's spell-checker.
replaceText(tpl, "{{name}}", "山田太郎");
replaceText(tpl, "{{date}}", new Date().toISOString().slice(0, 10));

// Replace across body + headers + footers + comments + footnotes:
replaceTextEverywhere(tpl, /\{\{(\w+)\}\}/g, (m) => values[m.captures[0]!] ?? "");

// Append a paragraph to the bottom of the body:
appendParagraph(tpl, "Updated by template fill on " + new Date().toLocaleString());

writeFileSync("out.docx", toUint8Array(tpl));
```

### Read text content

```ts
import {
  commentsPart,
  coreProperties,
  findTextEverywhere,
  openDocx,
  paragraphs,
  stylesPart,
  tables,
  text,
} from "@word-kit/core";

const doc = openDocx(bytes);

text(doc); // visible body text (joined paragraphs with \n)
paragraphs(doc); // WmlParagraph[]
tables(doc); // WmlTable[]
stylesPart(doc); // styles.xml AST (undefined if absent)
commentsPart(doc)?.comments; // each XML element of a <w:comment>
coreProperties(doc); // { title?, creator?, subject?, ... } from docProps/core.xml

// Inspect a template for placeholders before bulk-replacing:
for (const { partName, matches } of findTextEverywhere(doc, /\{\{\w+\}\}/g)) {
  console.log(
    partName,
    matches.map((m) => m.text),
  );
}
```

### Tracked changes

```ts
import { acceptAllRevisions, rejectAllRevisions } from "@word-kit/core";

acceptAllRevisions(doc); // keeps inserted text, drops deletions
rejectAllRevisions(doc); // drops insertions, keeps deletions as plain text
```

### Comments, footnotes, bookmarks, page breaks

```ts
import {
  addBookmark,
  addComment,
  addEndnote,
  addFootnote,
  addInternalHyperlink,
  appendPageBreak,
  paragraphs,
} from "@word-kit/core";

const para = paragraphs(doc)[0]!;

addComment(doc, para, { author: "Reviewer", text: "Please confirm." });
addFootnote(doc, para, "Source: internal report");
addEndnote(doc, para, "Glossary entry");

const bookmarkId = addBookmark(doc, "ch1", para);
addInternalHyperlink(doc, "ch1", "Jump to Chapter 1");

appendPageBreak(doc);
```

### Images

```ts
import { addImage } from "@word-kit/core";

const imageBytes = readFileSync("logo.png");
addImage(doc, imageBytes, {
  widthEmu: 914400, // 1 inch
  heightEmu: 914400, // 1 inch
  altText: "Company logo",
});
```

## Architecture in one diagram

```
@word-kit/core         ─── public Docx interface + standalone functions
       │
       ├── @word-kit/wml          ─── WordprocessingML AST + parsers + builders
       ├── @word-kit/ooxml-xml    ─── namespace-aware XML parser/serializer
       └── @word-kit/opc          ─── OPC (ZIP + Content Types + rels)
```

The lower layers are reachable via `doc.opc`, `doc.document`,
`stylesPart(doc)`, etc. for callers that need direct AST access.
